I believe that the step

∑1≤i1<…<ik≤nλi1,…,ik(⋀j=1kv∗ij)(u1,…,uk)=∑1≤i1<…<ik≤nλi1,…,ik(∏j=1kv∗ij(uj))∑1≤i1<…<ik≤nλi1,…,ik(⋀j=1kvij∗)(u1,…,uk)=∑1≤i1<…<ik≤nλi1,…,ik(∏j=1kvij∗(uj))
is wrong.

If this were true, for instance you would have

e∗1∧e∗2(e2,e1)=e∗1(e2)e∗2(e1)=0e1∗∧e2∗(e2,e1)=e1∗(e2)e2∗(e1)=0
instead of the expected
e∗1∧e∗2(e2,e1)=−e∗1∧e∗2(e1,e2)=−1.e1∗∧e2∗(e2,e1)=−e1∗∧e2∗(e1,e2)=−1.

So what you need to do is split the uiui before feeding them to the v∗ijvij∗.

The correct continuation from there is

∑λi1,…,ik(⋀j=1kv∗ij)(u1,…,uk)=∑λi1,…,ik(⋀j=1kv∗ij)(∑i=1nλ1ivi,…,∑i=1nλnivi)∑λi1,…,ik(⋀j=1kvij∗)(u1,…,uk)=∑λi1,…,ik(⋀j=1kvij∗)(∑i=1nλi1vi,…,∑i=1nλinvi)
Then use multilinearity and consider all ways to select one vivi at a time so as to have exactly the list {vi1,…,vin}{vi1,…,vin} in the end (with a possible minus sign depending on the parity of the permutation).

Note, however, that you don't have to actually do this. Only observe that it can be done, and that it can be done in the exact same manner for ff since it is multilinear and alternate as well, giving the same coefficients that depend on the matrix [λji][λij].
